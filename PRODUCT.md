# PRODUCT

## 高层目标
- 用一个本地脚本批量创建 Cloudflare Email Routing 规则，实现大量自定义地址自动转发到目标邮箱。
- 配置方式尽量简单（环境变量 + 可选输入文件），让零基础用户也能跑通。

## 主要用户情景
- 批量注册网站账号时，需要很多一次性/区分用途的邮箱别名。
- 业务团队需要为多个渠道或活动快速生成专用收件地址。
- 个人用户希望用大量自定义地址保护主邮箱隐私。

## 功能列表
- 读取环境变量配置（令牌、Zone、域名、目标邮箱等）。
- 支持两种地址来源：输入文件列表 / 前缀+数量批量生成。
- 调用 Cloudflare Email Routing API 创建规则（forward 动作）。
- 可选请求间隔，降低触发限流风险。
- 输出每个地址的创建结果与错误信息。

## 原子功能拆分
- 配置读取与校验。
- 地址列表生成与规范化。
- 规则请求体组装。
- 规则创建 API 调用。
- 结果输出与错误处理。

## MVP 范围
- 单域名、单目标邮箱、顺序创建。
- 仅支持 forward 动作（转发）。
- 失败不中断整体流程，逐条输出结果。

## 2026-03-09 Web UI 迭代
- 新增本地 Web 控制台，替代纯命令行入口。
- 用户可直接在页面中配置 Token、默认目标邮箱和批量生成参数。
- 用户可查看账号下全部域名，勾选需要投放 mail 的域名。
- 用户可在页面中完成规则查询、启停、删除和批量新增。
