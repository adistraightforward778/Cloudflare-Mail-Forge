# TASK

- [x] 任务 1：配置与输入约定
  - 目标：定义环境变量与输入文件规则。
  - 输入：Cloudflare 配置参数、输入文件路径。
  - 输出：可执行的配置清单。
  - 完成标准：文档中明确所有必填与可选项。
  - 风险点：令牌权限不足导致调用失败。

- [x] 任务 2：地址生成与规范化
  - 目标：支持文件列表与前缀生成两种模式。
  - 输入：本地部分/完整邮箱、域名、数量。
  - 输出：完整邮箱地址列表。
  - 完成标准：空行过滤且能生成正确格式地址。
  - 风险点：域名缺失导致拼接错误。

- [x] 任务 3：规则创建 API 调用
  - 目标：为每个地址创建 Email Routing 规则。
  - 输入：地址列表、目标邮箱、Zone ID、Token。
  - 输出：逐条创建结果与错误信息。
  - 完成标准：成功/失败均有可读输出，失败不中断。
  - 风险点：API 限流或目标邮箱未验证。

- [x] 任务 4：脚本交付与文档同步
  - 目标：交付可运行脚本并同步技术说明。
  - 输入：实现脚本与技术方案。
  - 输出：脚本文件与 TECH-REFER 文档。
  - 完成标准：脚本与文档描述一致。
  - 风险点：文档与实现偏差。

- [x] 任务 5：变更记录
  - 目标：记录新增与修改文件。
  - 输入：本次所有文件变更。
  - 输出：CHANGE.md 条目。
  - 完成标准：包含日期、文件、类型、影响。
  - 风险点：遗漏变更或字段不全。

- [x] 任务 6：本地 Web 服务
  - 目标：把 Cloudflare 能力封装成浏览器可调用的本地 API。
  - 输入：Token、Zone、Email Routing 规则接口。
  - 输出：`/api/config`、`/api/zones`、`/api/rules` 等接口。
  - 完成标准：首页和健康检查可访问，接口可返回 JSON。
  - 风险点：Token 权限不足或 Cloudflare 返回错误结构。

- [x] 任务 7：前端控制台
  - 目标：提供一页式 Web UI 管理域名和 mail 规则。
  - 输入：本地 API、默认配置、用户操作流。
  - 输出：配置、域名选择、批量创建、规则管理四个区域。
  - 完成标准：可在浏览器中完成查询 / 创建 / 删除 / 启停。
  - 风险点：前端状态与服务端状态不一致。
