# PLAN

## 阶段 1：需求固化与配置准备
- 目标：明确配置项、输入格式和 Cloudflare 前置条件。
- 功能：定义环境变量、输入文件规则、默认行为。
- 依赖关系：已有 Cloudflare Zone，Email Routing 已开启且目标邮箱已验证。
- 技术难点：权限范围正确、令牌可用性验证。
- 验收标准：给出清晰可执行的配置清单和输入说明。

## 阶段 2：脚本实现与核心流程
- 目标：完成地址生成与批量规则创建。
- 功能：从文件/前缀生成地址、构造请求体、调用 API、输出结果。
- 依赖关系：Cloudflare Email Routing API 可访问。
- 技术难点：API 错误处理、限流风险控制、不中断的失败容错。
- 验收标准：对 1-3 条样例地址可成功创建规则并输出结果。

## 阶段 3：验证与交付文档
- 目标：给出可复用的使用说明与项目文档。
- 功能：补齐产品/计划/技术/任务文档与变更记录。
- 依赖关系：脚本已可运行。
- 技术难点：文档与实现保持一致，避免配置歧义。
- 验收标准：文档齐全、与脚本一致，CHANGE 记录完整。

## 阶段 4：本地 Web 控制台
- 目标：把脚本能力收口为可访问的 Web UI。
- 功能：配置区、域名选择区、批量创建区、规则管理区。
- 依赖关系：Cloudflare Email Routing API 与 Zone API 可访问。
- 技术难点：本地安全存储、批量操作反馈、启停与删除的页面化管理。
- 验收标准：本机可通过浏览器访问页面并完成查询 / 创建 / 删除 / 启停。
